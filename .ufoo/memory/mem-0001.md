# Mutable scoped observer is intentional

MoLc Mutable<T> intentionally uses a scoped build observer to capture the currently building MutableWidget and register automatic subscriptions only while a MutableWidget builder is running. This relies on Flutter widget builds running synchronously on one isolate and is meant to provide GetX-like dependency tracking. Reads outside a MutableWidget builder are plain reads and must not subscribe stale widgets. Do not replace the observer with explicit subscriptions unless changing Mutable's core tracking semantics.
